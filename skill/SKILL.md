# vibe-animation

生成像素风 HTML 动画。一个文件，零依赖，双击即可运行。

## Trigger

`/vibe-animation <场景描述>`

示例：
- `/vibe-animation 让 Clawd 在夜晚大树下乘凉`
- `/vibe-animation 夏天海边，螃蟹在沙滩上玩球`
- `/vibe-animation 下雨天，Clawd 撑伞走在石板路`

---

## 工作流（五步，按序执行）

1. **读参考文件**：先读 `references/palette.json`（取色）和 `references/night_scene.html`（质量基准）
2. **设计场景**：确定背景、主角、1-2 个动态装饰元素，写出简要构思
3. **生成 HTML**：严格遵守下方的硬约束和质量标准
4. **自检**：对照 Quality Checklist 逐项确认，不通过则修改
5. **输出**：保存文件，用 `start <filename>` 在浏览器打开

---

## References 目录

```
skill/references/
├── palette.json      ← 官方色板，所有动画必须从这里取色
├── template.html     ← 起始模板，直接复制修改
└── night_scene.html  ← 质量基准，你的输出应达到这个水平
```

**使用原则**：读 `night_scene.html` 不是为了抄代码，而是感受节奏感、层次感和角色细节应有的密度。

## 硬约束（不可违反）

### 画布规格

```
外层 canvas：400×300 px（CSS 尺寸）
内层渲染分辨率：200×150（W=200, H=150）
缩放：ctx.drawImage(buf, 0, 0, 200, 150, 0, 0, 400, 300)
无抗锯齿：ctx.imageSmoothingEnabled = false（buf 和 ctx 都要设）
所有坐标：Math.floor() 取整，绝不出现半像素
```

### 像素风格

- 颜色：**≤ 16 种**，从 `palette.json` 取，不要随意生造
- 字体：不引入 Web 字体，文字用像素矩形画，或省略文字
- 随机数：使用固定种子生成器（保证每次刷新画面相同）

### 文件

- 零外部依赖（不引用任何 CDN / 图片 URL）
- 单 HTML 文件，双击浏览器可运行

## 动画质量标准

这是判断一个动画"好不好"的标准，不是可选的。

### 节奏感
- 每个动态元素的速度和频率要有**差异**（不能所有东西以同一节奏动）
- 背景元素慢（0.3-0.5 Hz），角色动作快（1-2 Hz），粒子最快
- 用 `Math.sin(t * speed + phase)` 给每个元素独立相位

### 表现力
- 角色有**主导动作**（最明显）+ **从属动作**（跟随主动作的细节）
  - 例：Clawd 身体上下浮动（主）→ 触角反向偏移（从）
- 表情/姿态要在小尺寸内清晰可辨（眼睛至少 2×2px）

### 循环设计
- 所有动画完美循环，不能有跳帧或停顿感
- 粒子（萤火虫、气泡等）用生命周期 + 重生机制，保持密度稳定
- 背景（草地、云）用偏移量 mod 宽度实现无缝滚动

### 层次感
- 绘制顺序：背景色 → 远景 → 中景树/建筑 → 粒子 → 角色 → 前景
- 远景颜色比近景**更暗更饱和度低**

## Clawd 角色规格

Clawd 是项目吉祥物（小螃蟹），在所有动画中保持一致造型。

```
基准尺寸（内部分辨率下）：
  身体宽：10px，身体高：7px
  整体高（含触角）：约 14px

颜色（从 palette.json 取）：
  body: #DE886D   claw: #C85C3A   belly: #F0D090   eye: #111111

必须包含：
  ✓ 双触角（相位相反地上下摆动）
  ✓ 双眼（2×2px black）
  ✓ 腹部高光（belly 色填充）
  ✓ 双钳（claw 色）
  ✓ 至少 3 条腿可见
```

如果场景不需要 Clawd，可自由设计其他角色，但需保持同等细节密度。

---

## 复杂度边界

### 合理范围内
- 1-2 个主角色（各自有 idle 动画）
- 1 个背景场景（天空/水面/室内等）
- 1-2 种粒子效果（萤火虫、气泡、落叶等）
- 天气效果（雨、雪）如果是场景核心主题

### 超出范围（不要做）
- 3 个以上独立角色
- 用户交互（点击、键盘）——除非用户明确要求
- 音频
- 复杂物理模拟（重力/碰撞）
